第三章：流程控制 (循环结构)

本章核心： 让程序学会“重复做某事”（转圈圈）。

1. for 循环

适用于已知循环次数的情况。这是最常用的循环。

  • 初始化：循环开始前执行一次（如 int i = 0）。

  • 条件判断：每次循环前检查，如果为真就继续，为假就停止（如 i < 10）。

  • 步进：每次循环结束后执行（如 i++，表示 i 加 1）。

  1. 必须使用 for 循环。

  2. 定义变量 sum 存储总和，必须初始化为 0。

预期输出结果：

答案：

易错：

  1. sum 忘记初始化（结果会是垃圾值）。

  2. 循环条件写错（如 i < 100 会少加一个 100）。

2. while 循环

适用于不知道具体次数，只知道停止条件的情况。

补充考点 (C 语言真假法则)： C 语言没有 true/false 关键字（C89标准）。

  • 0 是 假。

  • 非 0 都是 真。

  • while(1) 是死循环，while(0) 永远不执行。

3. do-while 循环 (导师补充)

(考点补充) 这是一个至少执行一次的循环。

  • 先干活，后判断。

  • 试卷考点：试卷 005.jpg 编程题第 3 题就用到了 do-while 结构。

4. 循环控制：break 与 continue

这是选择题必考点：

  1. break：彻底跳出循环（不干了，直接辞职）。

  2. continue：跳过本次循环剩余部分，直接进入下一次循环（今天请假，明天继续）。

5. 随机数生成 (Random Numbers)

这是一个进阶考点，通常用于游戏开发或模拟题。

三步走战略

  1. 头文件：#include <stdlib.h> 和 #include <time.h>。

  2. 播种：srand(time(NULL)); (在 main 开头写一次)。

  3. 生成：rand() % N (生成 0 到 N-1 的数)。

万能公式： 生成 [min, max] 之间的随机数：rand() % (max - min + 1) + min

习题 4 (对应：随机数生成)

题目： 编写一个“真·猜数字”游戏。

  1. 程序随机生成一个 1 到 100 之间的数字作为目标。

  2. 用户循环猜测，直到猜中。

要求： 必须正确包含头文件和播种代码。

预期输出结果： (每次运行的目标数字应不同)

答案：

易错：

  1. 忘记写 srand，导致每次运行生成的随机数一样。

  2. 取余公式写错，生成了 0 到 99 而不是 1 到 100。
